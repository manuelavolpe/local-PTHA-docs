Troubleshooting guide

Here is a non-exahustive list of issues that can be experimented on Mercalli@INGV during the process of installation and configuration of the Spack environment.

- Module not found when using pip

During the installation of the required Python packages in the Spack environment, you could get the message ModuleNotFoundError: No module named 'pip'.

If this is the case, you have to load py-pip before installing the Python packages with the command

foo@bar:$ spack load py-pip

- Python package not compatible with the current version of gcc

Usually Spack uses the (external) system compilers, and the installed version could conflict with the versions of the Python packages you are willing to install in the environment.

For example, numpy 2.3.4 is not compatible with gcc 4.8.5, and you could get the message ERROR: Problem encountered: NumPy requires GCC >= 9.3.

One possible way to solve this issue is to downgrade the numpy version, after checking which versions are available. For example:

foo@bar:$ pip index versions numpy

- Failures in building dependencies

A downgraded version of numpy can affect the installation of subsequent packages, as the last versions could require a more advanced version of numpy. As a consequence, they must be downgraded as well.

Here is a list of packages necessary for the workflow, that actually would require numpy 2.3.4:

  • netcdf4 1.7.3
  • h5py 3.15.1
  • cartopy 0.25.0
  • pandas 2.3.3

If instead numpy 2.2.6 has been installed, the issue can be solved as follows:

foo@bar:$ pip install netcdf4==1.7.2
foo@bar:$ pip install h5py==3.14.0
foo@bar:$ pip install contourpy==1.3.2 cartopy==0.24.1
foo@bar:$ pip install pandas==2.2.3

Once all the dependencies are satisfied, the command pip install -r /path-to-software/cheese-ptha-master/requirements.txt can be repeated to complete the installation.

- Problems when installing pyproj

When trying to install pyproj, you could get the message

x Getting requirements to build wheel did not run successfully.  
  │ exit code: 1
  ╰─> [1 lines of output]
      proj executable not found. Please set the PROJ_DIR variable. For more information see: https://pyproj4.github.io/pyproj/stable/installation.html
      [end of output]

The reason is that the package pyproj requires proj, which can be installed via pip as well, and on Mercalli a working combination is proj (v. 0.2.0) + pyproj 3.6.1:

foo@bar:$ pip install proj
foo@bar:$ pip install pyproj==3.6.1

However, this combination is not supported by Python 3.13 (currently the last version), and, to solve the issue, it could be required to re-create the Spack environment specifying a lower version of Python, for example 3.12.
